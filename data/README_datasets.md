# Dataset Provenance and External Validation Notes

This project uses a mix of tabular and ECG sources. For reproducibility and future external validation, keep a record of the exact dataset version, download date, source URL, and license.

## Current internal / processed assets

- `data/cardio.csv`: cardiovascular risk tabular dataset used for the tabular baseline.
- `data/hospital_admissions.csv`: hospital admissions dataset used for tabular/clinical features.
- `data/ptbxl_records/`: local PTB-XL ECG records or sample ECG artifacts, depending on your setup.
- `data/processed/`: preprocessed splits generated by `src/preprocess.py`.

## External ECG datasets to validate on

### PTBDB
- **Recommended use:** external ECG validation and transfer learning checks.
- **Expected format:** WFDB records (`.hea/.dat`) or pre-converted `.npy` files.
- **Downloader target:** `wfdb.dl_database('ptbdb', ...)` via `scripts/download_datasets.py --ptbdb`.
- **Provenance to record:** exact source, access date, label mapping, lead count, sampling rate, and any preprocessing performed.

### CPSC / challenge ECG sets
- **Recommended use:** cross-site generalization and robustness checks.
- **Expected format:** WFDB records or `.csv` / `.npy` exports.
- **Downloader target:** `wfdb.dl_database('cpsc2018', ...)` via `scripts/download_datasets.py --cpsc`.
- **Provenance to record:** challenge name, dataset split, access terms, and label source.

## Preparation workflow

Use `scripts/prepare_external_ecg.py` to standardize a local external ECG dataset into:

- `data/external/external_ecg_<dataset>_signals.npy`
- `data/external/external_ecg_<dataset>_labels.npy`
- `data/external/external_ecg_<dataset>_manifest.csv`

Example:

```bash
.venv/bin/python scripts/prepare_external_ecg.py \
  --input-root /path/to/ptbdb_or_cpsc \
  --dataset-name ptbdb \
  --labels-csv /path/to/labels.csv \
  --ecg-len 2000 \
  --out-dir data/external
```

## Notes

- If labels are not available, the script stores `-1` in the label array.
- The script tries to infer labels from folder names such as `positive`, `negative`, `normal`, and `abnormal`.
- For WFDB records, install `wfdb` in the project environment before running the prep step.
- Keep a copy of the original download instructions and any licensing constraints in this file when you add the real data.
